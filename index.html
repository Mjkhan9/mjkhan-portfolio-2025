MFA Incident Response Simulator

AWS IAM, CloudTrail, EventBridge, Lambda, DynamoDB, Python
View Project ‚Üí

An event-driven MFA incident detection pipeline using CloudTrail and EventBridge to identify authentication failures and abnormal login patterns in enterprise IAM environments. The system implements assisted remediation as an intentional safety constraint, requiring operator approval before IAM changes to limit blast radius and preserve accountability in regulated systems.

Architecture
EventBridge rules capture CloudTrail ConsoleLogin events, Lambda functions process incidents, DynamoDB stores incident state, and CloudWatch Insights queries enable post-hoc analysis
Safety Design
Assisted remediation workflow ensures operator approval before any IAM modifications, maintaining audit trails and preventing automated privilege escalation
Investigation Tools
Documented runbooks and CloudWatch Logs Insights queries for analyzing MFA failures, rate-limiting behavior, and escalation thresholds
CloudTrail EventBridge Lambda DynamoDB Python Terraform CloudWatch
