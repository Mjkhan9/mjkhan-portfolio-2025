## Context

The portfolio site needed a visual identity that:

1. Appeals to technical recruiters and engineering managers
2. Differentiates from generic portfolio templates
3. Reflects enterprise/infrastructure engineering domain
4. Works well for code-heavy content
5. Demonstrates frontend design sensibility

## Decision

Use a **dark-first design** with:
- Deep dark backgrounds (`#0a0a0f`, `#12121a`)
- Electric cyan accent (`#00d4aa`)
- Purple secondary (`#a855f7`)
- Subtle grid background pattern
- Glow effects on interactive elements

## Rationale

### 1. Target Audience Alignment

IT professionals spend 8+ hours daily in dark-themed environments:
- VS Code (default dark theme)
- Terminal emulators
- AWS Console (dark mode)
- GitHub (dark mode)

A dark portfolio feels native to their workflow, not jarring like a bright marketing site.

### 2. Code Readability

Dark backgrounds with syntax highlighting:
```python
# Much easier to read code samples
def provision_user(username: str) -> dict:
    return iam.create_user(UserName=username)
```

Light backgrounds with dark text create more visual competition with code blocks.

### 3. "Engineering Credibility" Signal

The aesthetic signals:
- "This person builds infrastructure, not just marketing sites"
- "They understand terminal/dashboard UX patterns"
- "They care about developer experience"

### 4. Differentiation from AI Templates

Generic AI-generated portfolios typically feature:
- White/light backgrounds
- Purple gradients (the "Vercel purple")
- Inter or system fonts
- Rounded cards with shadows

Our design specifically avoids these patterns:

| Generic Template | Our Approach |
|------------------|--------------|
| Light background | Deep dark (`#0a0a0f`) |
| Purple gradient hero | Subtle cyan glow |
| Inter font | Plus Jakarta Sans + JetBrains Mono |
| Drop shadows | Border glow effects |

## Color System

### Primary Palette

```css
/* Enterprise dark theme */
--dark-primary: #0a0a0f;    /* Main background */
--dark-secondary: #12121a;  /* Card backgrounds */
--dark-tertiary: #1a1a24;   /* Elevated surfaces */

/* Accent colors */
--accent: #00d4aa;          /* Electric cyan - CTAs, links */
--secondary: #a855f7;       /* Purple - secondary highlights */
--aws-orange: #ff9900;      /* AWS brand color for AWS content */
```

### Color Application Rules

| Element | Color | Reason |
|---------|-------|--------|
| Background | `#0a0a0f` | Deep but not pure black (reduces eye strain) |
| Cards | `#12121a` | Slight elevation from background |
| Primary CTA | `#00d4aa` | High contrast, action-oriented |
| Links | `#00d4aa` | Consistent with CTA |
| AWS badges | `#ff9900` | Brand recognition |
| Borders | `rgba(255,255,255,0.08)` | Subtle definition |

## Typography Choices

### Plus Jakarta Sans (Body)

Selected over Inter because:
- Less ubiquitous (Inter is default for many AI tools)
- Slightly more geometric character
- Excellent weight range (300-800)
- Good x-height for readability

### JetBrains Mono (Code)

Selected for:
- Designed specifically for coding
- Clear character distinction (0 vs O, 1 vs l)
- Ligature support for => and similar
- Free and open source

## Visual Effects

### Subtle Grid Background

```css
.sl-markdown-content::before {
  background: 
    linear-gradient(90deg, rgba(0, 212, 170, 0.03) 1px, transparent 1px),
    linear-gradient(rgba(0, 212, 170, 0.03) 1px, transparent 1px);
  background-size: 50px 50px;
}
```

This evokes:
- Engineering blueprints
- Terminal grid layouts
- Infrastructure diagrams

### Glow on Hover

```css
.project-card:hover {
  border-color: rgba(0, 212, 170, 0.3);
  box-shadow: 0 0 40px rgba(0, 212, 170, 0.15);
}
```

The glow effect:
- Provides satisfying interaction feedback
- Reinforces the "tech" aesthetic
- Creates depth without heavy shadows

## Accessibility Considerations

### Color Contrast

| Element | Contrast Ratio | WCAG Level |
|---------|---------------|------------|
| Body text (`#f1f5f9`) on dark | 15.8:1 | AAA ✅ |
| Accent (`#00d4aa`) on dark | 8.2:1 | AAA ✅ |
| Muted text (`#94a3b8`) on dark | 7.1:1 | AAA ✅ |

### Dark Mode Only

The site is dark-mode only because:
1. Light mode would require maintaining two complete themes
2. Target audience prefers dark mode
3. Consistent brand identity

## Consequences

### Positive

- Strong visual identity that stands out
- Excellent code block readability
- Appeals to technical audience
- Consistent with tools they use daily
- No theme toggle complexity

### Negative

- May not appeal to non-technical recruiters
- Print/PDF export needs special handling
- Screenshots on light backgrounds look stark
- Some users may prefer light mode (rare in target audience)

## Implementation Files
