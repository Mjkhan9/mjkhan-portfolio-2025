## Context

The portfolio needed a documentation-focused framework that:

- Renders MDX with code highlighting and custom components
- Supports dark mode out of the box
- Generates static HTML for GitHub Pages deployment
- Has built-in navigation, search, and table of contents
- Allows custom styling without fighting the framework

## Decision

Use **Astro Starlight** as the documentation framework for the portfolio site.

## Alternatives Considered

| Framework | Pros | Cons |
|-----------|------|------|
| **Docusaurus** | Large community, MDX support, versioning | React dependency, heavier bundle, opinionated structure |
| **VitePress** | Fast HMR, Vue-based, clean default theme | Vue ecosystem lock-in, less flexible styling |
| **GitBook** | Beautiful UI, no code required | Hosted service, limited customization, subscription cost |
| **Next.js + Nextra** | Full React power, API routes | Overkill for static docs, longer build times |
| **Starlight** ✅ | Zero-JS default, MDX, Tailwind-ready | Newer framework, smaller community |

### Why Starlight Won

1. **Zero client-side JavaScript by default** — Perfect for static portfolio where interactivity is minimal
2. **Native Tailwind CSS integration** — No extra configuration needed for custom theming
3. **Built-in dark mode** — Matches the enterprise dark theme requirement
4. **Astro's island architecture** — Can add React/Vue components if needed without full framework overhead
5. **GitHub Pages native support** — Simple deployment configuration

## Consequences

### Positive

- **Performance**: Lighthouse scores consistently 95+ due to minimal JavaScript
- **Customization**: Full control over CSS through Starlight's theme tokens
- **Developer Experience**: Hot reload, TypeScript support, familiar MDX syntax
- **SEO**: Static HTML output with proper semantic markup
- **Cost**: Free hosting on GitHub Pages

### Negative

- **Community Size**: Smaller than Docusaurus, fewer third-party plugins
- **Learning Curve**: Astro's content collections require understanding
- **Mermaid Integration**: Required manual configuration (see ADR-002)

## Implementation Notes

### Key Configuration Decisions

```javascript
// astro.config.mjs - Critical settings
export default defineConfig({
  site: 'https://mjkhan9.github.io',
  base: '/mjkhan-portfolio-2025',  // Required for GitHub Pages subpath
  integrations: [
    starlight({
      title: 'Mohammad Khan',
      customCss: ['./src/styles/custom.css'],  // Theme overrides
      // ...
    }),
    tailwind({ applyBaseStyles: false }),  // Preserve Starlight defaults
  ],
});
```

### Theme Token Overrides

Starlight uses CSS custom properties for theming:

```css
:root {
  --sl-color-accent: #00d4aa;  /* Electric cyan */
  --sl-color-black: #0a0a0f;   /* Deep dark background */
  --sl-font: 'Plus Jakarta Sans', system-ui, sans-serif;
}
```
