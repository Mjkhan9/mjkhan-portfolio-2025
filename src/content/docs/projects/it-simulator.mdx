## Key Engineering Decisions

### ADR-006: Zero Framework Architecture

**Context**: Modern web development typically uses React, Vue, or similar frameworks.

**Decision**: Build with pure HTML/CSS/JavaScript.

**Rationale**:
- **Demonstrates fundamentals** â€” Shows deep understanding of web standards
- **Zero bundle size** â€” Entire app is under 250KB
- **No build step** â€” Works offline, instant deployment
- **Longevity** â€” No framework deprecation concerns

**Trade-offs**:

| Pros | Cons |
|------|------|
| âœ… Full control over rendering | âŒ More manual DOM manipulation |
| âœ… No dependency vulnerabilities | âŒ No component hot-reloading |
| âœ… Works offline | âŒ More verbose code |
| âœ… Instant deployment | |

### ADR-007: Microsoft Fluent Design Inspiration

**Decision**: Use a dark theme inspired by VS Code and Windows Terminal aesthetics.

**Rationale**: IT professionals spend 8+ hours daily in dark-themed tools. The familiar aesthetic creates immediate comfort and credibility with technical recruiters.

## Code Highlights

### Ticket Generation Engine

The ticket engine uses weighted randomization to generate realistic scenarios:

```javascript
// ticketEngine.js - Realistic ticket generation
const ticketDatabase = {
  issues: [
    {
      category: "Software - Microsoft Office",
      priority: "High",
      description: "Microsoft Outlook is not launching...",
      relatedScripts: ["Reset-OutlookProfile", "Repair-OfficeInstallation"],
      keywords: ["outlook", "crash", "won't launch"]
    },
    // 10+ additional scenarios covering VPN, Teams, WiFi, Printers, etc.
  ]
};

function generateNewTicket() {
  const issue = ticketDatabase.issues[
    Math.floor(Math.random() * ticketDatabase.issues.length)
  ];
  
  return {
    id: `INC-2024-${Math.floor(Math.random() * 90000) + 10000}`,
    user: ticketDatabase.users[Math.floor(Math.random() * ticketDatabase.users.length)],
    device: ticketDatabase.devices[Math.floor(Math.random() * ticketDatabase.devices.length)],
    ...issue,
    status: "Open",
    createdDate: new Date().toLocaleDateString()
  };
}
```

### Animated Workflow Steps

Each workflow step includes visual feedback and state management:

```javascript
// flowEngine.js - Step animation with state tracking
function executeNextStep() {
  if (currentWorkflowStep >= workflowSteps.length) {
    completeWorkflow();
    return;
  }
  
  const step = workflowSteps[currentWorkflowStep];
  step.status = 'active';
  updateStepUI(step);
  
  // Simulate execution with realistic timing
  setTimeout(() => {
    step.status = 'completed';
    updateStepUI(step);
    currentWorkflowStep++;
    setTimeout(() => executeNextStep(), 500);
  }, step.duration);
}
```
